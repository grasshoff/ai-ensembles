7 CLI Execution

This chapter covers the command-line tools ainotebook and aiaudio for processing AI notebooks and audio files.

7.1 AINotebook CLI

The ainotebook command processes AI notebooks and manages visualizations.

7.1.1 Basic Commands

# Process a single notebook
ainotebook process notebooks/01-analysis.qmd

# Process multiple notebooks
ainotebook process notebooks/*.qmd

# Generate visualizations
ainotebook visualize notebooks/01-analysis.qmd

7.1.2 Command Options

ainotebook [command] [options] <files>

Commands:
  process    Process AI blocks in notebooks
  visualize  Generate visualization notebooks
  clean      Clean process directory
  info       Show notebook information

Options:
  --overwrite       Force reprocessing of blocks
  --model <name>    Override default model
  --parallel        Enable parallel processing
  --verbose        Show detailed output

7.1.3 Processing Examples

  1. Basic Processing
# Process with default settings
ainotebook process notebooks/01-analysis.qmd

# Force reprocessing
ainotebook process --overwrite notebooks/01-analysis.qmd

# Use specific model
ainotebook process --model claude-3-5-haiku notebooks/*.qmd
  1. Batch Processing
# Process all notebooks
ainotebook process notebooks/

# Process with pattern
ainotebook process "notebooks/*-analysis.qmd"

# Parallel processing
ainotebook process --parallel notebooks/*.qmd

7.2 AIAudio CLI

The aiaudio command handles audio file processing and transcription.

7.2.1 Basic Usage

# Transcribe audio file
aiaudio transcribe sources/audio/lecture.mp3

# Process with speaker detection
aiaudio transcribe --speakers sources/audio/interview.mp3

# Batch processing
aiaudio transcribe sources/audio/*.mp3

7.2.2 Command Options

aiaudio [command] [options] <files>

Commands:
  transcribe  Transcribe audio files
  diarize     Detect and label speakers
  clean       Remove temporary files

Options:
  --language <code>   Source language
  --speakers         Enable speaker detection
  --format <type>    Output format (txt/json)
  --timestamps      Include timestamps

7.3 Configuration

7.3.1 Environment Setup

# Required environment variables
export ANTHROPIC_API_KEY="your-key"
export GOOGLE_API_KEY="your-key"
export GROQ_API_KEY="your-key"

# Optional settings
export AI_NOTEBOOK_CACHE_DIR="/path/to/cache"
export AI_NOTEBOOK_TEMPLATE_DIR="/path/to/templates"

7.3.2 Settings File

# settings.json
{
  "paths": {
    "process_dir": "process",
    "template_dir": "sources/templates",
    "notebooks_dir": "notebooks"
  },
  "models": {
    "defaults": {
      "text": "claude-3-5-haiku",
      "audio": "whisper-large-v3"
    }
  },
  "processing": {
    "parallel": true,
    "cache": true,
    "overwrite": false
  }
}

7.4 Error Handling

7.4.1 Common Issues

  1. Authentication Errors
# Check API keys
ainotebook check-auth

# Test specific provider
ainotebook check-auth --provider anthropic
  1. Processing Errors
# Show detailed error information
ainotebook process --verbose notebooks/01-analysis.qmd

# Retry failed blocks
ainotebook retry notebooks/01-analysis.qmd

7.4.2 Logging

# Enable debug logging
ainotebook --debug process notebooks/*.qmd

# Save logs to file
ainotebook --log process.log process notebooks/*.qmd

7.5 Best Practices

  1. CLI Usage
    • Use appropriate commands for tasks
    • Check command options
    • Monitor processing status
  2. Error Management
    • Check authentication before processing
    • Use verbose mode for debugging
    • Keep logs for troubleshooting
  3. Performance
    • Enable parallel processing when appropriate
    • Use caching for repeated operations
    • Clean cache periodically
  4. Security
    • Secure API keys
    • Use environment variables
    • Validate input files

# CLI Execution

This chapter covers the command-line tools `ainotebook` and `aiaudio` for processing AI notebooks and audio files.

## AINotebook CLI

```bash
ainotebook [command] [options] <files>

  --overwrite       Force reprocessing of blocks
```

### Processing Examples

1. Basic Processing
```bash
# Process with default settings
ainotebook process notebooks/01-analysis.qmd
```
