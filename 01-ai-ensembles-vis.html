3 AI Ensembles Overview

AI Ensembles are collections of AI-powered notebooks that work together to process and analyze documents, audio, and other data sources. This chapter introduces the core concepts and architecture of AI ensembles.

3.1 Core Concepts

An AI ensemble consists of:

  1. AI Notebooks
    • Collection of .qmd files
    • Each notebook has a specific purpose
    • Notebooks can reference each other’s results
  2. Processing Pipeline
    • Document parsing
    • AI model processing
    • Result generation
    • Visualization creation

3.2 Directory Structure

ai-ensembles/
├── notebooks/           # AI notebook collection
│   ├── 01-analysis.qmd
│   └── 02-results.qmd
├── process/            # Processing results
│   ├── analysis.json
│   └── results.json
├── ai-views/           # Generated visualizations
│   └── analysis-vis.qmd
├── sources/           # Shared resources
│   └── templates/
│       └── document.j2
└── settings.json      # Configuration

3.3 Configuration

AI ensembles use a central settings.json file:

{
  "paths": {
    "process_dir": "process",
    "template_dir": "sources/templates",
    "notebooks_dir": "notebooks"
  },
  "models": {
    "defaults": {
      "text": "claude-3-5-haiku",
      "experimental": "gemini-2-thinking",
      "audio": "whisper-large-v3"
    }
  }
}

  2. AI Processing

    ainotebook process notebooks/*.qmd

  3. Visualization

5.2 Best Practices

  1. Notebook Organization
    • Use numbered prefixes (01-, 02-, etc.)
    • One main task per notebook
    • Clear dependencies between notebooks
  2. Template Management
    • Store shared templates in sources/templates
    • Use consistent variable names
    • Document template requirements
  3. Error Handling
    • Validate inputs before processing
    • Check for missing dependencies
    • Log processing errors
