4 AI Notebooks

AI Notebooks are Quarto documents (.qmd) that combine regular markdown content with special AI processing blocks. This chapter explains their structure and usage.

4.1 Supported Models

4.1.1 Anthropic Models

  • Gemini 2 (gemini-2)
    • Fast responses
    • Cost-effective
    • Best for: Quick analyses, drafts, iterations
  • Claude 3 Sonnet (claude-3-5-sonnet)
    • Balanced performance
    • Advanced reasoning
    • 200K context
    • Best for: Complex analysis, academic writing

4.1.2 Google Models

  • Gemini Pro
    • General purpose
    • Code generation
    • 32K context
    • Best for: Technical documentation, code analysis
  • Gemini 2.0 Flash Thinking (gemini-2.0-flash-thinking-exp-01-21)
    • Experimental model
    • Fast responses
    • Best for: Quick iterations and experiments

4.1.3 Groq Models

  • Mixtral 8x7B (mixtral-8x7b-32768)
    • 32K context window
    • High performance
    • Best for: Long-form analysis
  • LLaMA2 70B (llama2-70b-4096)
    • 4K context window
    • General purpose
    • Best for: Standard text processing

4.1.4 Other Models

  • O3-Mini
    • Lightweight model
    • Fast processing
    • Best for: Simple tasks and testing

4.1.5 Audio Processing Models

  • Whisper Large v3
    • Multiple languages
    • Robust transcription
    • Speaker diarization
    • Best for: Lecture transcription, interviews

4.2 Basic Structure

An AI notebook consists of:

  1. YAML Header
  2. Regular Markdown Content
  3. AI Processing Blocks
  4. Generated Results

4.2.1 Example: Document Analysis

---
title: "Document Analysis"
author: "Researcher Name"
date: last-modified
---

# Analysis Overview



### Example: Audio Processing


## AI Block Components

### 1. Model Selection

```yaml
# Text Analysis
model: gemini-2                           # Fast, efficient
model: claude-3-5-sonnet                          # Balanced performance
model: gemini-pro                                 # General purpose
model: gemini-2-thinking                         # Experimental Flash Thinking
model: mixtral-8x7b-32768                        # High performance
model: llama2-70b-4096                           # General purpose
model: o3-mini                                   # Lightweight

# Audio Processing
model: whisper-large-v3                          # High accuracy transcription

4.2.2 2. Processing Options

# Basic options
overwrite: false         # Skip if results exist
label: analysis_1        # Reference label
preamble: "text"         # Prepend to prompt

# Template usage
template:
  - file: analysis.j2    # Template file
    variables:           # Template variables
      TEXT: input.txt
      STYLE: academic

# Model-specific options
options:
  temperature: 0.1       # Response creativity
  max_tokens: 8192       # Maximum response length
  language: "en"         # Content language

4.2.3 3. File Assignments

files:
  document: sources/text.txt     # Load file content
  context: sources/context.txt   # Additional context
  audio: sources/lecture.mp3     # Audio file

4.3 Template System

Templates support multiple formats and use cases:

4.3.1 1. Academic Analysis Template

{# academic_analysis.j2 #}
Analyze the following text in {{STYLE}} style:

Text to analyze:
{{TEXT}}

Provide analysis focusing on:
1. Main themes
2. Key arguments
3. Supporting evidence

4.3.2 2. Data Extraction Template

{# data_extract.j2 #}
Extract structured information from this text:
{{TEXT}}

Required fields:
- dates: List all dates in ISO format
- names: Extract person names
- locations: Identify place names
- events: List key events

Output format: JSON

4.4 Advanced Features

4.4.1 1. Chain Processing

chain:
  - model: gemini-2
    label: metadata
  - template: analyze.j2
    variables:
      CONTEXT: "{{metadata}}"
  - template: summarize.j2

4.4.2 2. Parallel Processing

parallel:
  - label: text_analysis
    model: gemini-2
  - label: code_review
    model: gemini-pro
  - label: transcription
    model: whisper-large-v3

4.4.3 3. Conditional Processing

conditions:
  - if: "{{word_count}} > 1000"
    use: summarize.j2
  - else:
    use: direct_analysis.j2

4.5 Result Storage

Results are stored in JSON format with detailed metadata:

{
  "block_id": "analysis_1",
  "model": "gemini-2",
  "timestamp": "2024-02-15T14:30:00Z",
  "input": {
    "prompt": "Analyze this document...",
    "files": {
      "document": "path/to/source.txt"
    },
    "options": {
      "temperature": 0.1,
      "max_tokens": 8192
    }
  },
  "response": {
    "text": "Analysis results...",
    "tokens": 1250,
    "duration": 3.5,
    "model_version": "gemini-2-20240215"
  }
}

4.6 Cross-References

4.6.1 1. Within Same Notebook

---
model: gemini-pro
---
Review this analysis: {{analysis_1}}

4.6.2 2. Across Notebooks

---
model: gemini-2
---
Compare:
1. {{01-extract:metadata}}
2. {{02-analyze:findings}}

4.7 Error Handling

  1. Missing Files
files:
  required: true           # Fail if missing
  document: input.txt     
  fallback: default.txt   # Use if primary missing
  1. Processing Errors
