6 Data Structure

This chapter explains how data is organized in AI ensembles, including source files, templates, and processing results.

6.1 Directory Structure

ai-ensembles/
├── notebooks/           # AI notebook collection
├── sources/            # Source files and templates
│   ├── templates/      # Jinja2 templates
│   ├── documents/      # Input documents
│   ├── audio/         # Audio files
│   └── instructions/   # Task instructions
├── process/           # Processing results
├── ai-views/          # Generated visualizations
└── docs/             # Generated documentation

6.2 Source Files

6.2.1 1. Templates Directory

Templates are stored in sources/templates/ and follow Jinja2 syntax:

{# sources/templates/document.j2 #}
Analyze the following text in {{STYLE}} style:

Text to analyze:
{{TEXT}}

Required analysis:
1. {{#each REQUIREMENTS}}
   - {{this}}
   {{/each}}

6.2.2 2. Document Sources

Text documents in sources/documents/ can be referenced in notebooks:

files:
  input: documents/manuscript.txt
  context: documents/background.md
  metadata: documents/metadata.json

6.2.3 3. Audio Sources

Audio files in sources/audio/ support various formats:

files:
  recording: audio/lecture.mp3
  interview: audio/interview.wav
  meeting: audio/meeting.m4a

6.3 Process Directory

The process/ directory stores processing results and metadata:

6.3.1 1. Result Structure

{
  "notebook": "01-analysis.qmd",
  "blocks": [
    {
      "id": "block_1",
      "model": "claude-3-5-haiku",
      "input": {
        "files": {
          "document": "sources/documents/text.md"
        },
        "template": "analysis.j2",
        "variables": {
          "STYLE": "academic"
        }
      },
      "output": {
        "text": "Analysis results...",
        "metadata": {
          "tokens": 1250,
          "duration": 3.5
        }
      }
    }
  ]
}

6.3.2 2. Cache Organization

process/
├── cache/                    # Cached responses
│   ├── claude/              # By model
│   │   └── responses/
│   └── whisper/
│       └── transcripts/
├── metadata/                 # Processing metadata
└── results/                 # Final results

6.4 AI Views

Generated visualizations in ai-views/:

6.4.1 1. View Structure

ai-views/
├── _quarto.yml              # Quarto configuration
├── index.qmd                # Main page
├── analysis-vis.qmd         # Analysis visualization
└── results-vis.qmd          # Results visualization

6.4.2 2. View Configuration

# ai-views/_quarto.yml
project:
  type: book
  output-dir: ../docs

book:
  title: "AI Ensemble Results"
  chapters:
    - index.qmd
    - analysis-vis.qmd
    - results-vis.qmd

format:
  html:
    theme: sandstone
    code-fold: true

6.5 File Naming Conventions

  1. Notebooks

    NN-purpose.qmd          # e.g., 01-analysis.qmd

  2. Templates

    purpose_type.j2         # e.g., analysis_academic.j2

  3. Source Files

    YYYY-MM-DD_name.ext     # e.g., 2024-02-15_manuscript.txt

  4. Results

    notebook_id_block_id.json  # e.g., 01-analysis_block1.json

6.6 Best Practices

  1. Source Organization
    • Use clear directory structure
    • Follow naming conventions
    • Include metadata files
  2. Template Management
    • Group related templates
    • Document variables
    • Version control templates
  3. Result Storage
    • Organize by notebook
    • Include processing metadata
    • Regular cleanup of cache
  4. File Access
    • Use relative paths
    • Check file existence
    • Handle missing files gracefully

# Data Structure

This chapter explains how data is organized in AI ensembles, including source files, templates, and processing results.

## Directory Structure

```bash
ai-ensembles/
├── notebooks/           # AI notebook collection
├── sources/            # Source files and templates
│   ├── templates/      # Jinja2 templates
│   ├── documents/      # Input documents
│   ├── audio/         # Audio files
│   └── instructions/   # Task instructions
├── process/           # Processing results
├── ai-views/          # Generated visualizations
└── docs/             # Generated documentation
```

## Source Files

### 1. Templates Directory

Templates are stored in `sources/templates/` and follow Jinja2 syntax:

```jinja
{# sources/templates/document.j2 #}
Analyze the following text in {{STYLE}} style:

Text to analyze:
{{TEXT}}

Required analysis:
1. {{#each REQUIREMENTS}}
   - {{this}}
   {{/each}}
```

### 2. Document Sources

Text documents in `sources/documents/` can be referenced in notebooks:

```yaml
files:
  input: documents/manuscript.txt
  context: documents/background.md
  metadata: documents/metadata.json
```

### 3. Audio Sources

Audio files in `sources/audio/` support various formats:

```yaml
files:
  recording: audio/lecture.mp3
  interview: audio/interview.wav
  meeting: audio/meeting.m4a
```

## Process Directory

The `process/` directory stores processing results and metadata:

### 1. Result Structure

```json
{
  "notebook": "01-analysis.qmd",
  "blocks": [
    {
      "id": "block_1",
      "model": "claude-3-5-haiku",
      "input": {
        "files": {
          "document": "sources/documents/text.md"
        },
        "template": "analysis.j2",
        "variables": {
          "STYLE": "academic"
        }
      },
      "output": {
        "text": "Analysis results...",
        "metadata": {
          "tokens": 1250,
          "duration": 3.5
        }
      }
    }
  ]
}
```

### 2. Cache Organization

```bash
process/
├── cache/                    # Cached responses
│   ├── claude/              # By model
│   │   └── responses/
│   └── whisper/
│       └── transcripts/
├── metadata/                 # Processing metadata
└── results/                 # Final results
```

## AI Views

Generated visualizations in `ai-views/`:

### 1. View Structure

```bash
ai-views/
├── _quarto.yml              # Quarto configuration
├── index.qmd                # Main page
├── analysis-vis.qmd         # Analysis visualization
└── results-vis.qmd          # Results visualization
```

### 2. View Configuration

```yaml
# ai-views/_quarto.yml
project:
  type: book
  output-dir: ../docs

book:
  title: "AI Ensemble Results"
  chapters:
    - index.qmd
    - analysis-vis.qmd
    - results-vis.qmd

format:
  html:
    theme: sandstone
    code-fold: true
```

## File Naming Conventions

1. Notebooks
   ```
   NN-purpose.qmd          # e.g., 01-analysis.qmd
   ```

2. Templates
   ```
   purpose_type.j2         # e.g., analysis_academic.j2
   ```

3. Source Files
   ```
   YYYY-MM-DD_name.ext     # e.g., 2024-02-15_manuscript.txt
   ```

4. Results
   ```
   notebook_id_block_id.json  # e.g., 01-analysis_block1.json
   ```

## Best Practices

1. Source Organization
   - Use clear directory structure
   - Follow naming conventions
   - Include metadata files

2. Template Management
   - Group related templates
   - Document variables
   - Version control templates

3. Result Storage
   - Organize by notebook
   - Include processing metadata
   - Regular cleanup of cache

4. File Access
   - Use relative paths
   - Check file existence
   - Handle missing files gracefully
